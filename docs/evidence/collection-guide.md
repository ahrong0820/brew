# 근거 자료 수집 가이드

## 기본 흐름

1. 원본 자료를 Source로 등록한다.
2. 자료 위치를 확인한다.
3. 한 주장마다 Observation을 하나씩 만든다.
4. 조건, 변수, 결과와 한계를 기록한다.
5. 검수 전에는 `draft`로 유지한다.
6. 검수된 관찰만 활성 규칙에 연결한다.

## 필수 위치 정보

- 논문: 페이지, 절, 표 또는 그림
- 영상: 시작 시점, 필요하면 종료 시점
- 대회: 공식 영상 시점 또는 공식 문서 위치
- 제조사 자료: 제품 모델, 문서 버전, 공식 출처

## 분류 원칙

대회 레시피와 순위는 사용 사례다. 통제 비교 실험으로 분류하지 않는다. 자료가 직접 설명한 내용과 분석자의 추론을 구분한다.

## 입력 품질

- 숫자에는 단위를 적는다.
- 검수 완료 관찰에는 검수자와 검수일을 적는다.
- 적용 조건과 일반화 한계를 적는다.
- 원문 전체를 저장하지 않고 요약을 사용한다.
- 짧은 인용은 25단어 이내로 제한한다.

## 검증 명령

```bash
node --experimental-strip-types scripts/validate-evidence.mjs
```
