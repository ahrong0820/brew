# 근거 자료 수집 가이드

## 기본 흐름

1. 원본 자료를 Source로 등록한다.
2. 자료 위치를 확인한다.
3. 한 주장마다 Observation을 하나씩 만든다.
4. 조건, 변수, 결과와 한계를 기록한다.
5. 검수 전에는 `draft`로 유지한다.
6. 검수된 관찰만 활성 규칙에 연결한다.

## 필수 위치 정보

- 논문: 페이지, 절, 표 또는 그림
- 영상: 시작 시점, 필요하면 종료 시점
- 대회: 공식 영상 시점 또는 공식 문서 위치
- 제조사 자료: 제품 모델, 문서 버전, 공식 출처

## 공식 전문가 영상

전문가 본인의 공식 채널 영상은 1차 전문가 자료로 인정한다. 자막이 없다는 이유만으로 Source 등록을 보류하지 않는다. 다만 Observation은 타임스탬프, 챕터, 화면 레시피 카드 또는 제작자가 작성한 설명란처럼 직접 확인 가능한 위치가 있어야 한다.

채널 소유 관계, 검증 방법, YouTube 영상 ID를 기록한다. 재업로드나 제3자 요약은 공식 원본과 같은 직접 근거로 취급하지 않는다. 세부 기준은 [공식 전문가 영상 근거 정책](./expert-video-policy.md)을 따른다.

## 분류 원칙

대회 레시피와 순위는 사용 사례다. 통제 비교 실험으로 분류하지 않는다. 자료가 직접 설명한 내용과 분석자의 추론을 구분한다.

## 입력 품질

- 숫자에는 단위를 적는다.
- 검수 완료 관찰에는 검수자와 검수일을 적는다.
- 적용 조건과 일반화 한계를 적는다.
- 원문 전체를 저장하지 않고 요약을 사용한다.
- 짧은 인용은 25단어 이내로 제한한다.

## 검증 명령

```bash
node --experimental-strip-types scripts/validate-evidence.mjs
```
