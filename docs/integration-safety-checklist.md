# 통합 점검 및 데이터 충돌 방지

## 자동 보호 항목

### 추천 타이머 중복 실행

- 진행 중인 추천 추출 세션이 있으면 새 세션 생성을 차단한다.
- 같은 추천 조건을 5초 안에 연속 실행하면 두 번째 실행을 차단한다.
- UI가 빠르게 두 번 눌리더라도 저장 계층에서 중복 세션을 만들지 않는다.

### 원두별 추출 프로필

- `원두 + 드리퍼 + 그라인더 프로필 + 맛 방향` 조합은 하나의 프로필만 유지한다.
- 과거 데이터에 중복 프로필이 있으면 가장 최근 프로필을 남기고 세션을 해당 프로필로 연결한다.
- 존재하지 않는 원두를 참조하는 프로필과 세션은 제거한다.
- 세션의 원두 ID가 프로필의 원두 ID와 다르면 프로필 기준으로 복구한다.

### 현재 베스트와 최근 세션

- 각 프로필에는 현재 베스트가 최대 하나만 존재한다.
- 여러 세션이 `current-best`이면 최신 유효 베스트 하나를 남기고 나머지는 `good`으로 유지한다.
- `currentBestSessionId`와 `latestSessionId`가 삭제되거나 다른 프로필을 가리키면 다시 계산한다.

### 현재 베스트를 나만의 레시피로 저장

- 같은 베스트 세션을 반복 저장하면 새 레시피를 만들지 않는다.
- 같은 원두 조건에서 현재 베스트가 바뀌면 기존에 생성된 베스트 레시피를 갱신한다.
- 사용자가 변경한 레시피 이름과 기존 ID는 유지한다.
- 기존 `coffee-custom-recipes` 키와 배열 형식은 변경하지 않는다.

## 수동 확인 시나리오

1. 맞춤 추천에서 타이머 시작 버튼을 빠르게 두 번 누른다.
   - 추출 기록은 한 건만 생성되어야 한다.
2. 추출 진행 중 다시 맞춤 추천 타이머를 시작한다.
   - 진행 중 추출을 먼저 완료하라는 안내가 나와야 한다.
3. 같은 현재 베스트를 두 번 저장한다.
   - 나만의 레시피 수가 증가하지 않아야 한다.
4. 새 추출을 `좋음`으로 평가해 현재 베스트를 바꾼 뒤 다시 저장한다.
   - 기존 베스트 레시피가 최신 조건으로 갱신되어야 한다.
5. 원두를 삭제한 뒤 페이지를 다시 연다.
   - 해당 원두의 프로필·세션·진행 중 마커가 남지 않아야 한다.
6. 기존 나만의 레시피를 만든 상태에서 베스트 저장을 사용한다.
   - 기존 레시피 내용과 순서가 손상되지 않아야 한다.

## 범위

이번 점검은 단일 브라우저 저장소의 무결성과 일반적인 빠른 중복 입력을 대상으로 한다. 여러 탭에서 정확히 동시에 데이터를 수정하는 극단적인 경쟁 조건은 향후 데이터 백업·동기화 단계에서 별도로 다룬다.
